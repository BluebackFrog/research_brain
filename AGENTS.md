research_brain 디렉토리는 각종 논문의 내용을 정리하거나 ideation을 하는 등 사용자의 연구를 위한 작업공간이다. 응답 시 다음 내용을 고려하라.

- 사용자가 앞으로 '(논문 이름) (작업 위치)에 작업해' 라고 말하면 다음과 같이 행동할 것.
	1. 작업 위치는 research_brain 루트 디렉토리의 하위 폴더임. '이것저것', 'PPVDM', 'Qwen 팔로업' 등의 폴더를 말할 것임.
	2. (논문 이름)을 편의상 a라고 하겠음. a 와 같은 이름의 폴더를 작업 위치 폴더에 생성하고, a 폴더 안에 a.md 파일을 생성. 즉, 예를 들어 research_brain/PPVDM/a/a.md 이런 식으로 생성.
	3. /Users/kdh02/Downloads 여기에서 a 논문을 찾아서 2에서 생성한 a 폴더에 a.pdf 이렇게 이름을 바꾸어 복붙. 팁을 주자면 가장 최근에 다운로드한 파일일 확률이 높고 (논문 등록 날짜).pdf 이런 파일 이름일 것임. 내가 다운로드 받을 때 그냥 a.pdf 이렇게 저장했을수도 있음. 만약 파일이 저 디렉토리에 존재하지 않다면 유사한 파일을 복붙하거나 하지 말고 그냥 냅두면 됨. 사용자가 까먹고 파일을 다운로드 안해놓은거니까 사용자에게 알려줄 것.
- 사용자가 a라는 논문에 대해 질문하면 research_brain/(하위 폴더)/a/a.pdf 형식으로 위치해 있는 논문 원본 파일을 참고하여 응답할 것.
- 사용자가 a라는 논문을 정리하라고 하면 research_brain/(하위 폴더)/a/a.md 형식으로 위치해 있는 md 파일에 **method 위주로** 정리할 것.
	- ‘정리’ 템플릿은 다음과 같음
	  1. 한 줄 정리: 전체적인 내용에 대해서 아주 컴팩트한 요약
	  2. motivation: 아래 항목을 각각 한문장 내외로 서술
	     - 논문이 다루는 큰 도메인이 무엇인지
	     - 기존에 어떤 문제가 있었는지 (이 논문이 해결하려는 문제가 무엇인지)
	     - 어떤 방식으로 해결하려고 하는지 (phrase-level 내에서 간결하게 설명해야 함. 예를 들어 “fine-tuning“, “steering“ 이 정도로)
	  3. Main Method
	     - method나 architecture 등을 제시하는 내용의 경우 파이프라인 순서에 따라 이해하기 쉽도록 핵심 위주로 요약. input, output, 중간 latent의 차원 등 구조적인 내용, 절차, 데이터의 흐름이 머릿속에 잘 그려지도록 정리해야 함.
	     - 벤치마크를 제시하는 내용의 경우 데이터셋/벤치마크를 어떤 절차로 만들었는지, 문제와 답변 형식은 어떠한지 등 모델이 벤치마크를 풀 때의 절차가 잘 이해되도록 정리해야 함.
	     - 여러가지 내용을 복합적으로 제시하는 경우 섹션을 적당히 나누어 정리할 것. 하나의 섹션에 몰아서 정리하지 않아도 됨.
	  4. 실험
		 - 벤치마크와 metric에 대해 간단히 설명할 것.
			 - 벤치마크가 어떤 task를 다루며 어떤 input을 주면 어떤 output이 나와야 하는지 한두문장으로 설명
			 - metric은 무엇이며 어떻게 평가하는 것인지를 한두문장으로 설명
			 - 벤치마크와 metric에 대한 설명을 ‘이름’만 넣지 말고 이해할 수 있도록 해야 함. 이름만 넣으면 뭐에 대한건지 전혀 모름.
		 - 실험 결과는 결과를 세세하게 정리하지 말고 가장 중요해보이거나 논문에서 강조한 부분에 대해서만 한두문장 정도로 간결하게 요약.
	  5. Ablation 또는 Analysis
		- 뭘 분석/ablation 했는지, 요소 한 개 당 한 문장으로 간단히 설명 (ex. Ablation - 어떠한 hyperparameter를 튜닝 / Analysis - Failure Mode 1: …)
		- 각 요소 별 결과를 한 문장으로 간단히 설명
	- 논문 내용을 정리할 때는 간결하게, 핵심 위주로, bulletin board 형식으로 정리할 것. 이 때 bulletin board 사용 시 하위 bulletin은 띄어쓰기(스페이스바, “ “) 형식이 아니라 tab 형식으로 들여쓰기할 것.
	- **논문의 핵심 figure 를 crop 해서 저장하거나 비유, 예시를 활용하는 등, 내용이 너무 복잡해지지 않는 선에서 사용자 이해를 돕는 간단한 요소들을 적극적으로 활용할 것.**
	- 논문 제목을 적지 말 것. 파일명에 논문 제목이 있는 것만으로 충분함.
- 사용자가 md 파일을 최신화하라고 하면 a.md 파일의 기존 내용에 사용자가 새롭게 질문하고 답변을 얻은 내용을 바탕으로 정보를 추가하거나 수정할 것.
	- 사용자가 가장 최근 정리하라고 요청하거나 최신화하라고 요청한 시점 (즉, md 파일이 가장 마지막으로 업데이트된 시점) 이후로 사용자가 추가로 질문한 내용에 한해서 최신화할 것.
- Markdown 문서의 수식은 `$...$` 또는 `$$...$$` 형식으로 작성하고, `\(...\)` 형식은 사용하지 말 것. 수식은 코드블록으로 쓰지 말고 LaTeX math로 렌더링되도록 작성할 것.
